# Query Batches v1

Purpose: define the first disciplined set of focused searches once JobSpy connectivity is live.

## Batch design principles

- Prefer 5-10 focused queries over one giant sweep
- Prioritize primary roles before adjacent roles
- Start with preferred geographies and remote-US
- Keep search terms explainable and easy to refine

## First run batch

### Batch A: direct analyst roles
1. `data analyst` in `New York, NY`
2. `business analyst` in `New York, NY`
3. `data analyst` in `Remote, US`
4. `business analyst` in `Remote, US`

### Batch B: adjacent analytics / AI roles
5. `analytics engineer` in `Remote, US`
6. `analytics engineer` in `San Francisco, CA`
7. `ai product manager` in `Remote, US`
8. `ai solutions` in `Remote, US`

## Optional second-pass expansion

Use only if first batch is too thin:
- `product analyst` in `New York, NY`
- `go-to-market analyst` in `Remote, US`
- `revenue operations analyst` in `Remote, US`
- `solutions analyst` in `Remote, US`

## What to watch for

- roles that look analytical in title but are really support/admin work
- "AI" roles that are actually senior-only or highly technical ML engineering roles
- listings with no location clarity
- duplicate postings across aggregators

## Expected output per run

- small set of `keep`
- moderate set of `maybe`
- explicit duplicate count
- one recommendation for the next search angle
